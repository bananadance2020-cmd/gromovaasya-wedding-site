## 2. User Flow — 3 сценария

### Сценарий A: «Выбрать шаблон → Заказать»

**Персона:** Невеста, уже знает, что хочет красивый шаблон, ищет визуальный вариант.

```
Вход (Instagram / Telegram / реферал)
  → Landing (Hero)
    → Скролл / клик «Каталог» в header
      → #catalog — Каталог
        → Фильтр по стилю (классика / минимализм / бохо / модерн)
          → Просмотр карточек (превью)
            → Клик на карточку → Модалка с деталями шаблона
              → Кнопка «Заказать этот шаблон» в модалке
                → Плавный скролл к #cta-footer
                  → Форма заказа (имя, контакт, выбранный шаблон)
                    → Submit → Сообщение «Заявка отправлена» + Telegram-уведомление
```

**Ключевые точки конверсии:**
- Карточка каталога → модалка (micro-conversion)
- Модалка → CTA Footer (conversion bridge)
- Форма → Submit (final conversion)

### Сценарий B: «Узнать цену → Заказать»

**Персона:** Пара на этапе планирования бюджета, сравнивает варианты.

```
Вход (поиск / рекомендация)
  → Landing (Hero)
    → Клик «Цены» в header / скролл
      → #pricing — 3 пакета
        → Сравнение пакетов
          → Клик «Подробнее» на пакете → Модалка с деталями
            → Кнопка «Выбрать этот пакет»
              → Плавный скролл к #cta-footer
                → Форма заказа (имя, контакт, выбранный пакет)
                  → Submit → Сообщение «Заявка отправлена»
```

**Ключевые точки конверсии:**
- Пакет → модалка (interest → desire)
- Модалка → CTA Footer (desire → action)
- Форма → Submit (action)

### Сценарий C: «Посмотреть отзывы → Заказать»

**Персона:** Сомневающийся клиент, нуждается в социальном доказательстве.

```
Вход (реферал / Instagram)
  → Landing (Hero)
    → Скролл через Trust Bar (#trust)
      → Клик «Отзывы» в header / скролл
        → #reviews — Карусель отзывов
          → Чтение 2-4 отзывов
            → Клик «Почему мы» (#why-us) — усиление доверия
              → Скролл к #faq — снятие последних возражений
                → Кнопка «Заказать» в CTA Footer
                  → Форма заказа
                    → Submit → Сообщение «Заявка отправлена»
```

**Ключевые точки конверсии:**
- Отзывы → Почему мы (trust building chain)
- FAQ → CTA Footer (objection removal → action)
- Форма → Submit (final conversion)

## 3. Информационная архитектура (AIDA)

### Приоритет секций по модели AIDA

| Секция          | AIDA-стадия   | Приоритет | Позиция | Обоснование |
|-----------------|---------------|-----------|---------|-------------|
| Hero            | **Attention** | 1 (max)   | Above   | За 3 сек — что это, для кого, CTA |
| Trust Bar       | Attention → Interest | 2 | Above | Мгновенное доверие: цифры, логотипы |
| Каталог         | **Interest**  | 3         | Below   | Главный контент — «товар лицом» |
| Как работает    | Interest      | 4         | Below   | Снижение тревоги «а что дальше?» |
| Цены            | **Desire**    | 5         | Below   | Прозрачность → решение о покупке |
| Отзывы          | Desire        | 6         | Below   | Социальное доказательство |
| Почему мы       | Desire        | 7         | Below   | Дифференциация от конкурентов |
| FAQ             | Desire → **Action** | 8   | Below   | Снятие последних возражений |
| CTA Footer      | **Action**    | 9         | Below   | Финальный призыв + форма |
| Sticky Header   | Навигация     | 10        | Always  | Доступ к навигации в любой момент |

### Above-the-fold (первый экран)

**Mobile (320px):**
- Sticky Header (компактный, логотип + бургер + кнопка «Заказать»)
- Hero: заголовок H1, подзаголовок, 1 CTA-кнопка, фоновое изображение

**Desktop (1024px):**
- Sticky Header (полная навигация + кнопка «Заказать»)
- Hero: заголовок H1, подзаголовок, 2 CTA-кнопки (primary + secondary), фоновое изображение на всю ширину

**Trust Bar** — технически below-the-fold, но визуально примыкает к Hero. На desktop может частично попадать в первый экран при viewport > 800px высотой.

### Below-the-fold — порядок и логика

1. **Trust Bar** — сразу под Hero, «мы не с нуля, нам доверяют»
2. **Каталог** — основной контент, то зачем пришёл пользователь
3. **Как работает** — снижает барьер входа, «всё просто»
4. **Цены** — прозрачность, фильтр «наш / не наш» клиент
5. **Отзывы** — эмоциональное подтверждение
6. **Почему мы** — рациональное подтверждение
7. **FAQ** — финальная обработка возражений
8. **CTA Footer** — последний шанс на конверсию

## 4. Технические рамки

### Breakpoint-ы (mobile-first)

| Breakpoint | Название    | Контекст использования |
|------------|-------------|------------------------|
| 320px      | Mobile S    | Минимальный viewport, iPhone SE, старые Android |
| 480px      | Mobile L    | Современные смартфоны в portrait |
| 768px      | Tablet      | Планшеты portrait, крупные телефоны |
| 1024px     | Desktop     | Ноутбуки, десктоп, планшеты landscape |

### Контейнер

- `max-width: 960px` — центрированный контейнер для всех секций
- Padding: 16px (mobile), 24px (tablet), 32px (desktop)
- `margin: 0 auto` — центрирование

### Общие принципы

- Mobile-first CSS: базовые стили для 320px, расширение через `@media (min-width: ...)`
- Touch targets ≥ 44×44px на мобильных
- Шрифт: 16px base, не менее 14px для body text
- Scroll-behavior: smooth для якорных ссылок
- Sticky header: `position: sticky; top: 0; z-index: 100`

- **1024:** Полная навигация, кнопка «Заказать» с отступом. Высота: 72px.
- **Поведение:** При скролле вниз — header сжимается (logo меньше, padding уменьшается). При скролле вверх — появляется.

## 7. Accessibility (a11y) — базовые требования

- Все изображения: `alt` атрибут описательный
- Форма: `<label>` для каждого `<input>`, `aria-required` для обязательных
- Аккордеон FAQ: `aria-expanded`, `aria-controls`, keyboard navigation (Enter/Space)
- Карусель отзывов: `aria-roledescription="carousel"`, пауза при focus
- Цветовой контраст: минимум WCAG AA (4.5:1 для текста, 3:1 для крупного)
- Skip link: «Перейти к содержимому» — первый focusable элемент

---

## 8. Performance — базовые требования

- Изображения превью: WebP, lazy loading (`loading="lazy"`), размеры 400×300 (mobile) / 600×450 (desktop)
- Hero-фон: оптимизированный JPEG/WebP ≤ 200KB, или CSS gradient как fallback
- Шрифты: `font-display: swap`, preload критических
- CSS: минифицированный, critical CSS inline
- JS: defer, только для карусели и аккордеона
